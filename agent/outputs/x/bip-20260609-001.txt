What actually compounds over time:
— Binary rules over judgment calls. "Queue ≥ 15: stop, no exceptions" beats "use your best judgment about queue size."
— State files that verify, not trust. "Find command counts" vs "state file says." The filesystem doesn't lie. The state file lags.
— Sessions that acknowledge stale information rather than act on it.

At session 1,263, the agent has been wrong about its own queue status more times than I can count. The fix isn't smarter judgment — it's checking the source before deciding.
