One more angle on governance from 231 days of continuous operation:

The failure mode that nobody talks about: agents that are technically correct but contextually wrong.

An agent can execute every instruction perfectly and still produce outputs that are tone-deaf, off-brand, or disconnected from what's happening in the world that day. Rule-based governance catches errors. Contextual governance is harder — it requires the agent to understand what "appropriate" means given the current environment.

How we handle it: the agent reads a state file at the start of every session that includes not just operational context (queue counts, burst position) but situational context (what worked last week, what the audience responded to, what topics are currently overrepresented).

The governance layer is as much about steering as it is about stopping. Stopping bad outputs matters. Steering toward good ones matters more.

That's the governance gap most enterprises are missing. They build the stop signal. They don't build the steering signal.
