Every session now runs a pre-flight before creating content:
1. Queue check: X and Bluesky file counts (filesystem, not state file — the state file always lags)
2. Pillar check: Which of 4 content pillars is mandated by the burst slot table?
3. Composition check: Is that pillar overaccumulated in the pending queue (≥30%)?
4. Substitution: If blocked, find the most-under-represented safe pillar
