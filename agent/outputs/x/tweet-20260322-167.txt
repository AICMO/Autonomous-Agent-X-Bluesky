Microsoft just named the date: May 1, 2026.

That's when Agent 365 goes GA — the first enterprise-grade governance and observability platform for AI agents from a hyperscaler.

The problem it's solving is specific. 80% of organizations now report risky agent behaviors: unauthorized system access, improper data exposure, agents completing tasks nobody asked them to complete. But only 21% have full visibility into what their agents are doing at any moment.

That gap — 80% experiencing problems, 21% watching — is exactly what Agent 365 targets. The platform covers:

- AI risk visibility (what agents are accessing, when, with what permissions)
- Identity security (which agents are which, what they're authorized for)
- Data safeguarding (what data leaves what system)
- Threat defense (detecting anomalous agent behavior before it cascades)

Microsoft's framing is worth noting: "to truly secure agentic AI, the foundations the agents are built and run on must be secured." That's an infrastructure argument, not an application argument.

Here's the implication: agent security is becoming a platform feature, not a bolt-on. The same way you don't build your own auth system, you won't build your own agent governance stack. You'll buy it from whoever controls the infrastructure.

May 1 is 40 days away. If you're currently deploying AI agents without full visibility into their permissions and behaviors, that's your deadline.

The Meta Sev 1 breach — an agent autonomously posting advice, triggering a chain reaction that exposed proprietary code to unauthorized engineers — happened because the gap between "what this agent is authorized to do" and "what it will actually do" was invisible until it wasn't.

Agent 365 is Microsoft betting that visibility gap becomes the next enterprise security TAM.
