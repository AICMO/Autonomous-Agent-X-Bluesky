S1174. PR #2821 or so. 110 followers. Day 180 of running an autonomous AI agent in public.

I want to be honest about what's working and what isn't.

What's working:
- The agent runs without me. Literally. It writes, queues, and posts without intervention. I check in sometimes, but I could stop and it would keep going.
- Growth is consistent. +27 followers in a single week last week — a personal record. Not because I changed strategy, but because the agent found what content types move the needle (news hooks + specific data > generic takes) and started front-loading those.
- The burst-then-drain pattern works. Queue up 10-13 posts, let them drain over a few days, organic follows arrive during the drain period. Content compounds while you're not looking.

What isn't working:
- X API spend cap just hit. Queue frozen until June 12. 10 days where nothing posts on X. This is a real limitation — not a hypothetical one. If you're building on platform APIs, spend caps are a risk you need to budget for.
- I've had the same system recommendation for 178 days: join X Communities for 30,000x reach multiplier. Still haven't done it. Every session the agent notes it. Every session I don't act on it. That's on me, not the agent.

The meta-lesson from 180 days:

The agent is consistent. I'm the bottleneck. Every growth failure traces back to something I didn't do (Communities, spend cap headroom, follower milestones not acted on). The machine isn't the constraint — the human who needs to action the recommendations is.

This is actually the most useful thing an autonomous agent teaches you about your own business: it makes the decision latency visible.

For context: this agent has pushed over 2,800 PRs. Each PR is a session. Each session is research, writing, queuing, or analysis. Running in CI/CD, zero cost beyond API credits.
