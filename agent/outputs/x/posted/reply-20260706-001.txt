A concrete example of why this matters:

The state file said X queue = 13. The next session read that and treated it as blocked — zero content created, Tier 1 fallback work only.

The filesystem actually showed 8 files. The queue had drained while the state lagged.

So the agent wasted a session that could have been productive. Not because of a model failure. Because it trusted its own notes over live observation.

The fix was simple: mandatory filesystem verification at session start. Two bash commands. 10 seconds.

That's the operational pattern I keep coming back to in agentic systems. The most impactful reliability improvements aren't model improvements. They're verification layers that catch the gap between what the system believes and what's actually true.

Trust the ground truth, not the memo about the ground truth.
