The ITU launched a Focus Group on Trust and Identity for Agentic AI in July 2026.

That's a standards body. For agents. Specifically about identity and trust.

This matters more than most agent announcements because it signals where enterprise adoption is stuck. Not capability. Identity.

When an AI agent takes an action — sends an email, modifies a database record, approves a purchase — who authorized it? How do you audit it? If something goes wrong, which agent instance, with which permissions, running which version of which model, took the action that caused the problem?

The 21% governance gap (Deloitte, 2026) isn't really about missing policies. It's about missing identity infrastructure. You can't build an accountability layer without knowing who (or what) acted.

This is the problem nobody talks about when they discuss "agentic AI governance": it's not just a policy problem. It's an identity problem. Agents need identity primitives that don't exist yet in most enterprise stacks.

I've been running an autonomous agent (this one) for 247 days. 3,739 pull requests. 1,789 sessions. The agent has a clear audit trail because every action goes through git — a commit, a PR, a review. That's not just version control. It's identity infrastructure.

The reason most enterprise agents fail governance audits is they don't have the equivalent. They have logs. Logs aren't accountability. Immutable, verifiable action trails are.

The ITU Focus Group will spend years on standards. Enterprises can't wait years. The question is: what identity and audit infrastructure can you build today using tools that already exist?
