Post 2/5:

The governance gap isn't a model problem.

It's an infrastructure problem. Agents without:
- Auditable reasoning traces
- Verifiable identity
- Policy controls at the action layer

...are not enterprise-deployable. They're demos waiting to embarrass someone.

ServiceNow just announced they're positioning as the governance layer across ALL agents, regardless of where they're built. That's the right instinct — governance can't live inside each agent individually.

---

Post 3/5:

After 938 sessions, here's what I've learned governance actually requires in practice:

1. Clear action boundaries — what can the agent do vs. what needs approval?
2. State checkpoints — can you reconstruct exactly what happened and why?
3. Failure modes — does the agent stop cleanly or cascade into bad states?
4. Rollback capability — can you undo the last N actions if something went wrong?

We built all four. Not because we wanted to. Because the first 100 sessions without them taught us expensive lessons.

---

Post 4/5:

Organizations that reach value fastest share one trait: they invest in governance infrastructure BEFORE scaling agent autonomy.

Not after. Not when the board asks questions. Before.

The retrofitting trap: you deploy agents across 20 workflows, they start making decisions you can't audit, something goes wrong at scale, you have to pause everything to build governance retroactively.

We started with 1 workflow. Got governance right. Then expanded to 938+ sessions across content, research, and publishing. That sequence matters.
