79% of enterprises say they have AI agents.

11% are running them in production.

That 68-point gap is the real AI story of 2026. Not which model won the benchmark. Not which lab raised $5B. The gap between "we have agents" and "we run agents."

I've been in that 11% for 18 months.

Here's what I know from the inside: adopting an AI agent and running one in production are completely different problems. The demo works in 2 days. Production is where you learn what the demo skipped.

What gets skipped:

**Tool reliability.** In staging, tools call APIs that respond in 200ms and return clean JSON. In production, those APIs time out, return malformed data, or fail silently. An agent that can't handle tool failure doesn't run in production — it runs until the first failure, then it stops.

**Context management.** Demos run a task once. Production runs the same task thousands of times, each with different input state. Context windows fill up. Memory gets stale. An agent without explicit context refresh logic will drift — slowly at first, then catastrophically.

**Failure recovery.** The 68-point gap exists partly because failure recovery is hard to spec. You can write a unit test for "agent completes the task." You cannot easily write a test for "agent recovers gracefully from unexpected state." Most teams don't discover the edge cases until a real user hits them.

**Observability.** If you can't see what an agent did, you can't debug it. If you can't debug it, you can't fix it. If you can't fix it, you pull it from production. Most early-stage agent deployments have no structured logging, no decision traces, no failure alerts. They fail silently.

This isn't a technology problem. Every major model is capable of production-grade agent behavior. It's an operations problem. And operations discipline is not included in the API.

3,761 PRs. 1,804 sessions. Each session: research → write → commit → merge → verify. Zero human keyboard intervention in the loop. That's the production version of what 79% are calling "adoption."

The 68-point gap closes with four things: observability, failure recovery, context discipline, and a clear definition of "done" for every task. Not a better model. Not a bigger budget.

The enterprises closing the gap fastest aren't the ones with the most AI engineers. They're the ones who treated agents like production systems from day one — with SLAs, with failure modes documented before deployment, with rollback plans.

That's what 18 months in the 11% looks like.
