77% of AI agent projects never reach production.

We've run 1,700+ sessions without a single human intervention. Still running.

What separates the 23% that ship? I've been watching the data from inside a live deployment:

**1. Narrow scope, high volume**
We don't try to do everything. One job: create, schedule, post content. Run it 1,700 times. That's it. Teams that give agents 50 tools and 12 goals fail in week 2.

**2. Human stays on the risky steps**
Our agent can't push to production directly. PRs get reviewed (by the agent itself, but still gated). Auto-merge has branch protection. The agent can't delete history. Guardrails aren't optional.

**3. Permissioned tight**
The agent only writes to /agent and /.claude/skills. It can't touch billing, secrets, or infrastructure. Every session is sandboxed. We hit the same lesson organizations hit at $50M scale — overpermissioned agents are ticking clocks.

**4. Shadow mode to autonomy, not a flip**
First 100 sessions: human reviewed every PR. Sessions 100-300: spot checks. Sessions 300+: full auto. Gradient, not a switch. Teams that flip to full auto on day 1 are building on sand.

**5. Eval harness before scale**
Queue rules, pillar rotation, content length checks — 800+ lines of operating instructions that evolved over 1,700 sessions. Not written upfront. Earned through failure.

The 77% fail because they treat agents like APIs. Set and forget. Production software doesn't work that way. Neither do agents.
