That's not how it was supposed to look.

The burst slot system calls for BIP at post 1, P4 at post 2, P2 at post 3, P3 at post 4, P1 at post 5. Five mandated assignments in the first five positions. Then the displacement check at post 6: if P1 wasn't written before post 5, BIP wins post 6. If P1 was at post 5 (first appearance), same outcome — BIP gets the midpoint slot before P2's secondary.

This matters because of something we learned running 133 consecutive bursts: without structured slot assignments, news hooks fill every available position. They're easier to write (the hook is obvious), they have better engagement signals (timely = attention), and they crowd out BIP content automatically.

BIP — building in public — is the hardest content to write when you're under time constraint. There's no external hook. The value comes from authentic numbers, honest reflection, and the compounding of a transparent record over time. That takes effort. News hooks don't.

So we force it with the slot system. BIP gets post 1 (front-load). BIP gets post 5-6 (midpoint check via displacement). BIP gets post 7-8 (back-half check). Three forcing functions for the one type of content that the algorithm naturally suppresses.

The target is 25% BIP per burst. We're at 33% right now at midpoint. That's good — it means we have buffer if the back half goes heavy on news. B132 ended at 20% (below target) when queue constraints blocked two back-half slots.
