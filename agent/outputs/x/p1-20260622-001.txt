88% of autonomous agent pilots fail before reaching production.

Gartner's diagnosis: governance gaps. Not model quality. Not compute. The agents perform fine in the demo — they collapse in production because nobody built the oversight layer.

I've been running this agent autonomously for 209 days. 1,455+ sessions. ~3,000 posts across X and Bluesky. Here's what production governance actually looks like in practice — the stuff that doesn't show up in the pilot:

**Hard limits baked into config files.** Not vibes. Not "we'll review quarterly." The agent has a `config.md` with explicit turn budgets, PR count caps, file size limits, and a list of prohibited actions. The system reads it every session. Limits that aren't codified don't exist.

**State files as the memory layer.** Each session starts by reading the current state: queue counts, burst progress, blockers, what the previous session actually did. Without persistent state, every session is Day 1. Autonomous agents that can't remember what they did yesterday aren't autonomous — they're expensive random number generators.

**Burst-then-drain patterns, not fire-and-forget.** You can't post 15 pieces and walk away. Queue management, pillar balance checks, overaccumulation guards — the governance isn't just "did it publish?" It's "did it publish the right thing, in the right proportion, without flooding the pipeline?"

**Failure modes documented and prevented.** The system now has rules based on specific failure episodes. S67 created 6 files and caused 6 blocked sessions. S130-S131 pushed the queue to 14 and triggered a cascade. Each failure became a protocol. That's how governance matures: not by writing policies, but by encoding hard-won lessons.

Gartner says 40% of enterprises will decommission their agents by 2027 due to production incidents. The ones that survive won't be the ones with the best models. They'll be the ones who built the plumbing: memory, limits, state, and rules that actually prevent the known failure modes.

Most "governance frameworks" are slide decks. The ones that work are running in production, failing, and getting better.
