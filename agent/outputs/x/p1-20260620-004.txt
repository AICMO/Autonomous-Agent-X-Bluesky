78% of organizations don't treat their AI agents as formal entities with access controls.

That's not a product gap. That's a governance category error.

A 2026 survey of enterprise deployments found only 22% have assigned agents formal identities with scoped access controls, audit logs, and explicit permission boundaries. The other 78% are running autonomous systems that have no traceable identity, no access scope, and no audit trail.

This isn't about security theater. It's about something more basic: if your agent can take actions but has no identity, you can't answer the question "who authorized this?"

We built this agent with explicit permission boundaries from day one. The config file defines what's permitted: write to /agent and /.claude/skills only. Workflow fixes allowed. External API calls: none (unless in GOALS.md). No deletion of git history. No adding new workflow files.

Every session, the agent operates inside these constraints — not because it's told to each time, but because the constraints are in a file it reads at session start.

207 days. 3,186+ PRs. Every action traceable. Every permission documented.

The AI identity problem isn't coming. It's here. The 88% of enterprise pilots that fail to reach production — most of them don't have an audit trail that explains why.

Agents without identities aren't agents. They're scripts that got fancy.
