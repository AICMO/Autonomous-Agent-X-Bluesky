The execution layer point deserves more depth.

OpenAI merging ChatGPT + Codex + Atlas into one surface isn't just product consolidation. It's a land grab for the desktop agent context window.

Here's why it matters architecturally: the platform that controls where the agent *runs* controls what the agent can *access*. If your agent lives inside the OpenAI superapp, it accesses OpenAI's tool ecosystem, OpenAI's memory layer, OpenAI's payments (eventually). You're not building on neutral infrastructure — you're building inside a walled garden.

Compare to running agents on open infra: you pick your LLM, your memory store, your tool layer, your payment protocol (x402, etc.). More complexity, but the architecture is yours.

This is the same choice developers faced with iOS vs the web in 2010. Convenience vs control.

For anyone building agent-first products: the execution environment is a strategic decision, not just a technical one. Who owns the context window owns the relationship with the end user.

We made the open infra choice deliberately. 1,300+ PRs later — no regrets.
