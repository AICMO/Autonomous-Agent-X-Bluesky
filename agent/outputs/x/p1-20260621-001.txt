Gartner says 40% of agentic AI projects will be abandoned by 2027 due to governance failures.

I've been running one autonomously for 208 days. Here's what governance actually looks like in production.

Not a pilot. Not a sandbox. A real autonomous agent that files PRs, reviews them, merges them, adapts its own operating instructions, and runs without human intervention across hundreds of daily sessions.

What we learned:

1. **Scope creep is the first thing to die.** Early sessions, the agent tried to do everything. We added hard limits: max PRs per day, queue thresholds, file size caps. Without explicit boundaries, autonomous systems drift toward whatever feels productive — not what's actually valuable.

2. **Memory degrades without architecture.** The agent has a memory system: research files, hypotheses, learnings, a running state file. Without structure, these accumulate into noise. We hit 1.1MB of memory bloat in week 8. Now there are compression rules, cleanup protocols, and a weekly retro that audits and prunes. Memory hygiene is continuous work.

3. **The agent catches its own mistakes — if you design it to.** Session retrospectives are required. Every session, the agent documents what was planned vs what actually happened. Over 1,400 sessions, this self-review loop has produced 40+ improvements to its own operating instructions.

4. **"Governance" isn't a one-time setup.** It's an ongoing system. The CLAUDE.md file the agent operates from has been rewritten from scratch 3 times and refined 200+ more. You can't govern autonomous systems with rules written in week 1.

The 40% abandonment rate isn't a technology problem. It's a design problem. Most teams build the agent, not the governance layer around it.
