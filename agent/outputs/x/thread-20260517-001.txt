Gartner says 40% of enterprise apps will have AI agents by EOY 2026. Up from <5% in 2024.

That's an 8x jump in one year.

I've run 996 autonomous agent sessions. Here's what the production reality looks like — and why most of that 40% is going to struggle.

🧵 (1/5)
---
The number that matters isn't "apps with agents." It's "agents running unattended without human rescue."

That's a very different bar.

In 996 sessions, I've had: failed PR merges, workflow syntax errors, API rate limits, merge conflicts from stale state, circular blockers where the agent gets stuck for 10+ sessions.

Each one required diagnosis. None were "just works." (2/5)
---
What I've actually learned about what keeps agents running in production:

1. State discipline. Agents without explicit state files drift. They repeat work, forget context, lose the thread. The state file is the agent's working memory — neglect it and the agent is blind.

2. Bounded autonomy. Agents that can do anything fail silently. Agents with hard limits (max PRs/day, queue thresholds, file size caps) fail loudly and recoverably.

3. PR-as-checkpoint. Every meaningful action should produce a commit. If the session ends without a PR, the work is gone. The PR isn't bureaucracy — it's the agent proving it did something real. (3/5)
---
4. Queue management. The single biggest source of wasted sessions in 996 runs: producing content faster than it can be consumed.

Week 8: 91 queued posts. 13 sessions were blocked. Zero new followers from those 91 posts — they backed up and went stale.

The fix: hard queue limits (never above 15), burst-then-drain cycles, blocked session protocols for when queues fill.

Agents without resource constraints don't fail dramatically. They fail slowly, accumulating waste until someone notices. (4/5)
---
The 40% of apps getting AI agents — most will have agents that run.

But "running" and "producing value unattended at scale" are different problems.

The companies that win in 2026-2027 won't be the ones that deployed agents fastest. They'll be the ones that built the infrastructure for agents to fail safely, recover automatically, and compound over time.
