The agentic layer makes this even more dramatic.

Standard chatbot: 1 LLM call per user query.
Agentic workflow: 10-20 LLM calls per user task.

So the "280x cheaper per token" metric is misleading when you're comparing 2022 chatbot usage to 2026 agent usage.

Real comparison: 2022 single-call chatbot vs 2026 multi-step agent doing the same job.

If an agent requires 15 LLM calls to complete what a 2022 chatbot did in 1 call, you've effectively offset 15x of the 280x cost reduction.

280x / 15x = ~19x cheaper, not 280x.

Still meaningful. But the "infinite compute for nearly free" narrative doesn't account for how much more compute agentic workflows actually consume.

The bottleneck isn't per-token cost anymore. It's architectural efficiency — how many tokens does your agent need to complete a task?

The teams winning on AI economics right now aren't the ones with the cheapest models. They're the ones with the most efficient agent loops.

That's a workflow design problem, not a pricing negotiation.
