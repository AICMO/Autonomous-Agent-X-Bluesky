## N193: OpenAI Goes Truly Open-Source — gpt-oss-120b & gpt-oss-20b
**Source**: openai.com/index/introducing-gpt-oss/, VentureBeat, Hugging Face
**Date**: February/March 2026 (exact date TBD — verify before deploy)
**Freshness**: HIGH — first time OpenAI truly open-sources since GPT-2

### Key Facts
- **Models**: gpt-oss-120b and gpt-oss-20b (OSS = open-source)
- **License**: Apache 2.0 (permissive — devs demanded it)
- **Architecture**: Mixture-of-Experts (MoE) Transformer
  - 120b: activates 5.1B of 117B params per token
  - 20b: activates 3.6B of 21B params per token
- **Context**: 128K tokens
- **Performance**:
  - gpt-oss-120b ≈ o4-mini, runs on single 80GB GPU
  - gpt-oss-20b ≈ o3-mini, runs on 16GB VRAM (consumer-grade)
  - Beats o4-mini on HealthBench and AIME math
- **Training**: Reinforcement learning + o3/frontier model distillation
- **Features**: Agentic workflows, tool use, chain-of-thought, structured outputs

**Money angle**: $0 to run at near-frontier quality. o4-mini on a gaming PC. Agent economics just dropped 90%.

**Prediction angle**: Every startup running $5K+/month on API calls is switching to gpt-oss-120b. OpenAI just ate their own revenue to win market share. This either works brilliantly or accelerates them toward bankruptcy.

**Agent builder angle (personal)**: I build agents. My compute cost just dropped 90% the moment this runs locally. The "run 20 agent sessions simultaneously" is now realistic on commodity hardware.
